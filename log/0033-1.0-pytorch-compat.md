### 함정 — Mojo native list literal → Python 함수 인자

```mojo
# ❌ 1.0에서 컴파일 에러
var a = torch.tensor([[1.0, 2.0], [3.0, 4.0]])
# error: could not convert element of 'args' with type 'List[List[Float64]]'
#        to expected type 'ConvertibleToPython & Copyable'

# ✅ Python.list() 명시 변환 필요
var row1 = Python.list(); row1.append(1.0); row1.append(2.0)
var row2 = Python.list(); row2.append(3.0); row2.append(4.0)
var data = Python.list(); data.append(row1); data.append(row2)
var a = torch.tensor(data)
```

→ Mojo `List[T]` *flat*은 ConvertibleToPython 가능하지만 *중첩 List*는 아님. 1.0에서도 미해소된 *interop edge case*.

## 5) 결론

- **PyTorch (CPU) 호환성 = *완전 동작***. torch + transformers 모두 정상.
- **GPU PyTorch는 본 머신 CPU only라 미검증** — `torch.cuda.is_available()=False` 확인.
- **함정**: 중첩 list literal 직접 전달 불가. Python.list() 명시 변환 필요 — *boilerplate*.
- **사용 시나리오**:
  - Mojo native 핫커널 + PyTorch 모델 wrapping: 가능 (Python interop 위)
  - PyTorch 모델 → Mojo로 layer 단위 교체: 가능 (work 0030 CNN과 결합)
  - PyTorch 학습 + Mojo inference: 표준 export 경로 (state_dict → safetensors → max.engine) 사용

후속:
- GPU 머신에서 `torch.cuda` 활용 + Mojo SIMT 코드와 zero-copy 결합 가능성
- ndarray ↔ torch.Tensor *zero-copy view* (`torch.from_numpy`) 비용 측정
- Mojo `parallelize` 핫커널을 PyTorch custom op으로 등록 (TorchScript / C++ extension 패턴)
