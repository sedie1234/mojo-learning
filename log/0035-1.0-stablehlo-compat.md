# work 0035 — [1.0] StableHLO 호환성

- **status**: done (negative result, 공식 입장 확인)
- **date**: 2026-05-11
- **plane**: MOJO-45

## 1) 무엇을 하고자 하는지

Mojo 1.0 + MAX 26.3에서 *StableHLO MLIR dialect*를 (a) 받아 컴파일 (b) emit (c) 직접 사용 가능한지 검증.

StableHLO는 OpenXLA의 표준 ML compute opset. PyTorch (Torch-MLIR), JAX, TensorFlow가 *공통 IR로 수렴*하는 형식 — *XLA → StableHLO → backend*.

→ Mojo가 StableHLO를 받는다면 *3개 메이저 프레임워크 모델을 모두* Mojo로 가져갈 수 있음 → MAX/Mojo 생태계의 *결정적 진입점*.

## 4) 실제 결과

### 4.1 Modular 공식 입장

`forum.modular.com/t/mlir-dialect-import-for-mojo/774` 정확 인용:

> "Mojo does not currently support importing external MLIR dialects"
> "[It is] an aspirational goal for Mojo as a native frontend for MLIR"
> "I'd love to get there, but one step at a time!" — Stef (Modular)

**No roadmap plans**.

### 4.2 차단 요인 (Stef 발언)

1. **Parser integration**: 외부 dialect 인식 메커니즘 없음
2. **MLIR linking**: 동일 MLIR에 link되는 두 라이브러리 동시 사용 불가
3. **Mixed dialect output**: 표준 Mojo 구문 (`if` 등)이 *native Mojo op*만 생성 → 사용자 코드가 dialect-specific op으로 직접 lowering 안 됨
4. **Lowering gap**: 사용자 dialect → executable로 lowering 미구현

### 4.3 가능한 모든 경로 부정 결과

| 경로 | 결과 | 비고 |
|------|:--:|------|
| `mojo build foo.mlir` (StableHLO `.mlir` 직접 빌드) | ❌ | mojo CLI는 `.mojo` 소스만 받음 |
| `mojo build --emit stablehlo` (Mojo → StableHLO emit) | ❌ | `--emit asm\|llvm\|object\|...` 외 dialect 옵션 없음 |
| `max compile foo_stablehlo.mlir` | ❌ | MAX는 HF safetensors + 등록 architecture만 |
| `from std.mlir import stablehlo` (코드에서 op 사용) | ❌ | std에 외부 dialect 노출 없음 |
| ONNX → StableHLO → MAX | ❌ | MAX는 StableHLO frontend 없음 (work 0015 결론) |
| JAX → StableHLO → Mojo native | ❌ | 동일 |

## 5) 결론

### 5.1 핵심 결론

**Mojo 1.0 / MAX 26.3은 StableHLO와 *호환되지 않음***. work 0014/0015/0025의 "MAX는 ONNX/PyTorch/TF/StableHLO 모두 미수용" 결론을 *forum 공식 입장*으로 재확인.

### 5.2 의미

- PyTorch/JAX/TF 사용자가 *Mojo 생태계 진입*하려면 *모델 재작성* 필요 (max.graph로 직접 작성 또는 HF safetensors + registered architecture)
- *진정한 다중 프레임워크 호환성*은 1.0/26.3 시점 *부재*
- Modular는 *MLIR 호환성 자체*를 *희망*하지만 *infrastructure 미준비*

### 5.3 다른 도구와의 비교

| 도구 | StableHLO 받음? |
|------|:--:|
| **MAX 26.3** | ❌ |
| IREE | ✅ (StableHLO frontend 1급) |
| OpenXLA backend (TF/JAX) | ✅ (StableHLO 표준) |
| onnxruntime | △ (ONNX → 일부 StableHLO 변환 가능 외부 도구) |

→ MAX는 *production LLM serving 경로*에서는 강하지만 *cross-framework 모델 import*에서는 IREE/onnxruntime 대비 한계.

### 5.4 우회 (실용)

1. **PyTorch → ONNX → onnxruntime** (work 0034 검증) — 표준
2. **MAX 등록 모델만 사용** — HF safetensors + registered architecture
3. **max.graph로 모델 *수동 변환***  (`ops.matmul`, `ops.conv2d` 등 호출로 다시 작성)
4. **Mojo native로 layer 단위 재구현** (work 0030 conv2d 패턴)
