### 4.2 Gemma4 - 1단 게이트 *여전히 차단*

```
transformers 5.2.0
```

→ 0.26 cycle 결론 그대로. transformers 5.2.0도 `gemma4` 미인식. Gemma4는 신규 모델로 transformers/MAX 모두 lag.

### 4.3 Qwen3 - encoding 매트릭스 통과, *더 깊은 곳에서 차단*

**bfloat16 + CPU**:
```
ValueError: The encoding 'bfloat16' is not compatible with the selected device type 'cpu'.
encodings available for this model: bfloat16, float32, float8_e4m3fn
```
→ 0.26과 동일하게 encoding × device 매트릭스에서 차단.

**float32 + CPU**:
```
graph build 단계 도달
→ qwen3/qwen3.py:139, Qwen3TransformerBlock.__init__:
    self.allreduce = Allreduce(num_accelerators=num_devices)
→ allreduce.py:59:
    self.devices = [Accelerator(id=id) for id in range(num_accelerators)]
ValueError: failed to create device: No supported "gpu" device available.
  CUDA information: Failed to open library "libnvidia-ml.so.1"
  HIP information: Failed to open library "libamdhip64.so"

```

→ **encoding gate는 통과, but 모델 정의 코드 *내부*에 `Accelerator(id=...)` 하드코드 의존** → CPU에선 graph build 자체가 실패.

### 4.4 차단 layer 비교 (0.26 vs 1.0)

```
[Stage 1]  transformers AutoConfig    [Stage 2]  MAX architecture registry    [Stage 3]  encoding × device    [Stage 4]  graph build
─────────────────────────────────     ─────────────────────────────────────   ──────────────────────────       ──────────
0.26 Gemma4:  ❌ blocked              N/A                                       N/A                              N/A
0.26 Qwen3:   ✓                       ✓                                         ❌ bfloat16+CPU rejected         N/A
1.0 Gemma4:   ❌ blocked (변경 없음)   N/A                                       N/A                              N/A
1.0 Qwen3:    ✓                       ✓                                         ✓ float32+CPU 가능               ❌ Accelerator hardcoded
```

→ **Qwen3의 차단 layer가 stage 3 → stage 4로 *깊어졌을 뿐*, 여전히 CPU에서는 inference 불가**.

## 5) 결론

- **MAX 26.3 + 1.0.0b1에서도 CPU LLM serving은 본질적으로 막혀 있음**. 0.26 cycle의 결론(`MAX는 GPU 중심`)이 *유지*.
- **차단 위치 변화**: 0.26은 *encoding gate*, 1.0은 *graph build 안 Accelerator 하드코드*. 즉 1.0에서 *encoding 매트릭스는 약간 개선*됐지만 *모델 정의 코드는 여전히 GPU 강제*.
- **Gemma4는 모든 점프에서도 차단** — transformers/MAX 모두 lag. 신규 모델에 *유의미한 시간 지연*이 있다.
- **새로 추가된 architectures**: Qwen3, Qwen3Moe, Qwen2.5-VL, Gemma3 family, Exaone, DeepseekV3.x 등 — *주요 LLM 패밀리* 등록은 빠르게 진행됨.
- **CPU 사용자에게**: 여전히 **llama.cpp 권장**. Qwen3-4B GGUF가 이미 `~/models/qwen-3/4B-Instruct-gguf-{bf16,Q4}/`에 준비됨 — llama.cpp 경로가 *실용 정답*.

**0.26 cycle 결론과 종합**:
> "MAX의 *진짜 가치*는 GPU + 등록된 production LLM에서 발현. CPU 학습/실험에선 llama.cpp/onnxruntime이 여전히 더 단순하고 안정적."

→ 1.0 검증으로 *재확인*. GPU 도착 후에 MAX 본격 검증이 옳은 결정.
