## 1) 무엇을 하고자 하는지

0.26 시점에 작성한 `fn main(): print(...)` 형태의 가장 단순한 hello world 코드가 1.0.0b1에서도 컴파일/실행되는지 확인. 기존 학습 자산의 회귀 가능성 파악이 목적.

## 2) 수행한 일

- `experiments/0016-1.0-hello-regression/01_hello_026.mojo` — `fn main():` 그대로 (0.26 스타일).
- `experiments/0016-1.0-hello-regression/02_hello_def.mojo` — `def main():` (1.0 스타일).
- 새 venv `.venv-1.0/`에 `pip install --pre mojo` → 1.0.0b1 설치.
- `mojo run`(JIT)과 `mojo build`(AOT) 둘 다 시도.

## 3) 예상되는 결과

- `fn` 코드는 deprecation warning과 함께 컴파일은 성공할 것 (release notes에 "deprecated, will be error in next release" 명시).
- `def` 코드는 warning 없이 성공.
- AOT 산출물 크기는 0.26 시절(44 KB) 근방.

## 4) 실제 결과

```
Mojo 1.0.0b1 (a9591de6)

=== 01_hello_026.mojo (fn 사용) — JIT ===
01_hello_026.mojo:2:1: warning: 'fn' is deprecated, use 'def' instead
fn main():
^~
def
Hello, Mojo!
Int = 42
```

- ✅ `fn`은 정확히 예상대로 deprecation warning 후 정상 실행.
- ✅ `def`은 warning 없이 깨끗하게 성공.
- 🆕 **AOT 산출물 크기 18.6 KB** (0.26의 44 KB 대비 **58% 감소**).

| 항목 | 0.26 | 1.0.0b1 |
|------|--:|--:|
| AOT hello world | 44 KB | **18.6 KB** |

`fn` warning 메시지는 커서까지 표시(`^~`)하고 자동 fix-it (`def`)도 제시하는 잘 만들어진 형태.

## 5) 결론

- **회귀 양호** — 가장 단순한 코드는 그대로 컴파일/실행됨. fn warning은 문법 직접 알려주는 형태.
- **AOT 크기 대폭 감소**는 예상 외 — 이유 추정: 1.0에서 `libNVPTX.so` 등 default GPU 의존을 줄였거나 stripping 정책 변화. `ldd` 검증 후속 work 후보.
- **Migration 비용 추정 (가장 단순 케이스)**: 0 — fn → def sed 한 줄 치환.
